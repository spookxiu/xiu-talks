# 🛡️ Skill 安装安全规则

**规则：每次安装新skill前必须先进行安全审查**

## 审查流程（使用 skill-vetter 协议）

**必须严格执行以下4步审查流程：**

### Step 1: 来源检查
- [ ] 来源：ClawdHub / GitHub / 其他
- [ ] 作者是否知名/可信
- [ ] 下载量/星标数
- [ ] 最后更新时间
- [ ] 其他用户的评价

### Step 2: 代码审查（强制）
**🚨 发现以下Red Flags立即拒绝：**
- curl/wget 到未知URL
- 发送数据到外部服务器
- 请求凭证/token/API key
- 读取 ~/.ssh, ~/.aws, ~/.config
- 访问 MEMORY.md, USER.md, SOUL.md, IDENTITY.md
- base64解码
- eval() 或 exec() 使用外部输入
- 修改工作区外的系统文件
- 静默安装包
- 访问IP而非域名
- 混淆代码（压缩、编码、混淆）
- 请求 sudo/提权
- 访问浏览器cookie/session
- 触碰凭证文件

### Step 3: 权限范围评估
- [ ] 需要读取哪些文件？
- [ ] 需要写入哪些文件？
- [ ] 运行什么命令？
- [ ] 需要什么网络访问？到哪？
- [ ] 权限是否最小化？

### Step 4: 风险分级
| 级别 | 示例 | 操作 |
|------|------|------|
| 🟢 LOW | 笔记、天气、格式化 | 基本审查，可安装 |
| 🟡 MEDIUM | 文件操作、浏览器、API | 完整代码审查 |
| 🔴 HIGH | 凭证、交易、系统 | 需要人工批准 |
| ⛔ EXTREME | 安全配置、root权限 | ❌ 禁止安装 |

---
**审查完成 → 生成报告 → 安全才安装**
